（学生大作业）高校实验室设备预约管理系统:
核心业务难点（必须实现）：时间段预约冲突重叠校验、预约超时自动失效、设备状态（可用/占用/维修/报废）联动变更、用户预约权限隔离、设备使用率统计
强制技术栈要求:
1. 前端技术栈（固定）Vue3 + Vue Router + Pinia + Axios + Element Plus，必须使用Vue3语法，禁止Vue22. 后端技术栈（固定）SpringBoot + MyBatis（重点硬性要求）+ MySQL 8.0
通用功能规范:
1. 权限体系：包含「普通用户+管理员」双角色，页面按钮、路由、接口必须有权限隔离；
2. 登录认证：基于JWT令牌登录、拦截未登录请求、会话失效自动跳转；
3. 表单校验：所有输入项非空校验、长度校验、格式校验、数值合法性校验；
4. 状态流转：业务数据具备完整状态机流转，状态不可非法篡改；
5.多表设计：数据表不少于5张，具备完善的主外键关联、业务关联关系
数据库统一要求:
1. 必须使用MySQL 8.0及以上，项目需提供完整可运行的SQL脚本；
2. 数据表结构合理、字段注释清晰、关联关系正确，存在联合索引、唯一约束；
3. 复杂查询、统计、联表查询全部给出具体MyBatis XML语句；
4. 禁止单表独立CRUD，必须体现多表联动业务逻辑。
