##  使用流程示例

### 示例1：完整的管理员工作流程

1. **登录系统**
   - 访问 http://localhost:8083/login
   - 输入管理员账号密码
   - 点击登录 → 自动跳转到Dashboard

2. **查看统计数据**
   - Dashboard显示用户、设备、预约统计
   - 查看最近预约记录
   - 点击快捷按钮进入对应管理页面

3. **添加新用户**
   - 进入"用户管理"页面
   - 点击"添加用户"
   - 填写用户信息 → 确定
   - 提示"添加成功，默认密码为：123456"

4. **管理设备状态**
   - 进入"设备管理"页面
   - 找到需要维修的设备
   - 点击"修改状态" → "设为维修"
   - 设备状态变为红色"维修"标签

5. **审批预约**
   - 进入"预约管理"页面
   - 切换到"待审核"标签
   - 找到待审批的预约
   - 点击"批准"或"拒绝"
   - 二次确认后完成审批

6. **退出登录**
   - 点击右上角用户名
   - 选择"退出登录"
   - 确认后返回登录页

## 💡 开发经验总结

### 成功经验

1. **模块化设计** - 每个功能独立成页面，易于维护
2. **组件复用** - Element Plus组件统一风格
3. **路由嵌套** - 管理员页面使用嵌套路由，结构清晰
4. **状态管理** - 使用ref和reactive管理响应式数据
5. **错误处理** - 统一的拦截器处理HTTP错误

### 遇到的挑战及解决方案

#### 挑战1: 路由守卫逻辑复杂
**问题**: 需要处理多种跳转场景  
**解决**: 
```javascript
// 需要登录的页面
if (to.meta.requiresAuth && !token) {
  next('/login')
} 
// 已登录用户访问登录页
else if (token && (to.path === '/login' || ...)) {
  next('/admin/dashboard')
} 
else {
  next()
}
```

#### 挑战2: 设备状态与预约审批联动
**问题**: 设备维修时不能批准预约  
**解决**: 
- 后端API已经实现检查逻辑
- 前端捕获错误并显示提示信息
- 用户友好地告知失败原因

#### 挑战3: 时间段转换显示
**问题**: 数据库存储的是数字（0-3），需要显示为时间范围  
**解决**: 
```javascript
const getTimeSlot = (time) => {
  const timeSlots = {
    '0': '8:00-10:00',
    '1': '10:00-12:00',
    '2': '14:00-17:00',
    '3': '19:00-21:00'
  }
  return timeSlots[time] || time
}
```

### 最佳实践

1. **代码组织** - 按功能模块划分目录（admin/）
2. **命名规范** - 语义化的变量和函数名
3. **注释清晰** - 关键逻辑添加注释
4. **API封装** - 统一的request工具类

## 📞 技术支持

### 常见问题FAQ

**Q: 如何查看某个用户的详细信息？**  
A: 当前版本在用户管理页面可以看到基本信息（ID、用户名、邮箱、生日）。后续版本会添加用户详情页面。

**Q: 设备使用频率是如何计算的？**  
A: 使用频率 = 该设备的批准预约数 / 所有设备的总批准预约数。点击"更新使用频率"按钮可重新计算。

**Q: 为什么有些预约无法批准？**  
A: 可能原因：
- 设备状态为"维修"或"报废"
- 设备在该时间段已被其他预约占用
- 网络错误或服务器异常

**Q: 数据会自动刷新吗？**  
A: 不会。需要手动刷新页面或重新加载数据。建议在操作后手动刷新查看最新数据。

**Q: 如何批量操作？**  
A: 当前版本暂不支持批量操作，需要逐个处理。后续版本可能会添加此功能。
