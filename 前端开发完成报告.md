## 🔗 API接口映射

### 已实现的接口

| 前端功能 | 后端接口 | 方法 | 参数 | 返回值 |
|---------|---------|------|------|--------|
| 管理员登录 | `/login/admin` | POST | username, password | token, userId, username, role |
| 管理员注册 | `/login/register/admin` | POST | id, username, email, birthday, password, adkey | null |
| 修改密码 | `/admin/profile/change-password` | POST | oldPassword, newPassword | null |

### 接口调用位置

1. **Login.vue** (第145-168行)
   ```javascript
   const response = await request.post('/login/admin', {
     username: loginForm.username,
     password: loginForm.password
   })
   ```

2. **Register.vue** (第216-227行)
   ```javascript
   await request.post('/login/register/admin', {
     id: registerForm.id,
     username: registerForm.username,
     email: registerForm.email,
     birthday: registerForm.birthday,
     password: registerForm.password,
     adkey: registerForm.adkey
   })
   ```

3. **UpdatePassword.vue** (第175-186行)
   ```javascript
   // 先登录获取token
   const loginResponse = await request.post('/login/admin', {
     username: passwordForm.username,
     password: passwordForm.oldPassword
   })
   
   // 再修改密码
   await request.post('/admin/profile/change-password', {
     oldPassword: passwordForm.oldPassword,
     newPassword: passwordForm.newPassword
   })
   ```

### 遇到的问题及解决方案

#### 问题1: ESLint组件命名警告
**原因**: Vue推荐组件名使用多词命名  
**解决**: 在vue.config.js中禁用lintOnSave，或在组件中添加eslint-disable注释

#### 问题2: PowerShell命令分隔符错误
**原因**: PowerShell不支持&&作为命令分隔符  
**解决**: 使用分号`;`代替`&&`

#### 问题3: 修改密码需要先登录
**原因**: 后端没有提供独立的忘记密码接口  
**解决**: 采用先登录获取token，再调用修改密码接口的方案

### 最佳实践
1. **代码组织** - 按功能模块划分目录结构
2. **命名规范** - 使用语义化的变量和函数名
3. **注释清晰** - 关键逻辑添加注释说明

## 📞 技术支持

### 常见问题FAQ

**Q: 为什么第三方登录按钮是禁用的？**  
A: 后端API中没有实现QQ、微信、支付宝登录功能，因此前端保留了按钮样式但设置为禁用状态。

**Q: 注册时的adkey是什么？为什么必须是11111？**  
A: adkey是管理员注册的密钥，用于防止普通用户随意注册为管理员。固定值"11111"在后端代码中硬编码验证。

**Q: 修改密码为什么要先登录？**  
A: 因为后端没有提供独立的"忘记密码"或"重置密码"接口，所以采用先验证身份（登录）再修改密码的方案。

**Q: 如何测试前端功能？**  
A: 
1. 确保后端服务运行在http://localhost:8080
2. 启动前端服务：`npm run serve`
3. 访问http://localhost:8083
4. 先注册管理员账号（adkey: 11111）
5. 使用注册的账号登录测试

**Q: 前端和后端端口冲突怎么办？**  
A: 可以修改vue.config.js中的devServer.port配置，改为其他端口如8084。
